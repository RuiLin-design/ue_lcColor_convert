Unreal Engine 5 / Localization

Going Global:
Localization Workflows & Dynamic Reflow

Felix / UI Technical Artist
Published Apr 9, 2026

In high-end UI development, hardcoding text is a critical error. A truly professional project must possess Cultural Awareness. When your UI switches from English to German or French, text length can expand by 30% to 200%. How do you keep buttons from breaking? How do you handle complex CJK characters? This guide will help you build an industrial-grade localization system.

Phase 1: Activating the Dashboard

Unreal Engine’s built-in Localization Dashboard is the central command for all translation management.

Action Checklist:

  • 01
    Gathering Text: Go to Tools -> Localization Dashboard. Click "Gather Text." The engine will scan all FText variables across your UMGs.

    Note: Always use FText over String; FText supports localization IDs and display caching.

  • 02
    Adding Cultures: Click "Add New Culture" to include target languages (e.g., German, Japanese).

Phase 2: Fixing "UI Overrun" via Dynamic Reflow

A common mistake is setting fixed widths for buttons. In a localized project, you must master Auto-Slot and Wrapping.

A

Enable Size To Content

For button components, enable Size To Content. This allows the button background to expand horizontally based on the translated text length rather than clipping it.

B

Automatic Wrapping

In dialogue boxes, enable Auto Wrap Text. This ensures that when a word in German is exceptionally long, it wraps to the next line instead of being cut off.

⚠️ Localization Secret: For Asian languages (CJK), you must configure a Composite Font. You can set Roboto for English and Noto Sans as a Fallback for Japanese in a single asset to solve "Tofu" (missing character) issues.

Phase 3: Real-time Language Switching

Don’t force players to restart! Implement a seamless language switch using the Blueprint API.

// Runtime Culture Switching (Blueprint/C++ API):
1. [Set Current Culture] (Target: "zh-Hans" or "de")
2. [Save Config] -> Ensures persistence on next launch
3. The system broadcasts an update; all FText widgets redraw instantly.

Phase 4: Managing Heavy Font Assets

In global projects, font files can be massive. You should use the UE5 Font Face system to isolate .ttf assets for optimized loading.

  1. Load on Demand: Use the Asset Manager to load that 20MB CJK font only when the user selects that specific language.
  2. Sub-font Families: Use Sub-font Family to set dedicated fallback paths for special symbols or emojis.

Golden Rules for Global UI

  • Never Concatenate Strings: Use the Format Text node for structures like "Level: {Num}" instead of using the "+" operator.
  • Buffer for Expansion: Always leave at least 20% visual "breathing room" in your UI containers to account for text expansion.
  • Respect Case Sensitivity: Some languages (like German) capitalize all nouns; avoid using ToLower in your logic.
  • Numerical Formatting: Many European countries use commas , as decimal separators. Let the localization system handle value display automatically.

Localization is more than just translation; it is an act of respect for global players. Master it, and your UI earns its ticket to the world stage.
